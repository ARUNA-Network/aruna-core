# ADR-0005: Hybrid PoW + PoS Consensus

## Status
Proposed

## Context
A decentralized public blockchain requires a consensus mechanism to secure the network, order transactions, prevent double-spending, and distribute rewards. There are two primary paradigms:
1. **Proof of Work (PoW):** Secure, mathematically objective, but favors hardware centralization (ASIC farms) and consumes significant energy.
2. **Proof of Stake (PoS):** Energy-efficient and provides fast finality, but is prone to plutocracy (large holders control validator selection) and long-range attacks.

ARUNA Network aims to enable mining on consumer devices (Android, Raspberry Pi, Mini PCs) while maintaining protocol security, fast transaction finality, and decentralization.

## Problem
Selecting a single consensus mechanism introduces major risks for ARUNA's vision:
1. **ASIC Centralization (Pure PoW):** If ARUNA uses pure PoW, industrial mining operations will eventually build ASICs that render consumer ARM/Android devices obsolete, destroying the decentralization target.
2. **Plutocracy (Pure PoS):** If ARUNA uses pure PoS, early pre-mined/founder coins or venture capital could purchase a majority of the stake, permanently locking out community governance and validation.
3. **51% Attack Susceptibility:** Low-hashrate CPU mining chains are vulnerable to hashpower renting (e.g., NiceHash), allowing attackers to rewrite history.

We need a consensus model that combines the security of PoW with the finality of PoS, protecting the chain from centralization and hashpower-only attacks.

## Decision
We adopt a **Hybrid Proof of Work (PoW) + Proof of Stake (PoS)** consensus model.

### Key consensus rules and parameters:
1. **Block Production Pipeline:**
   * Block headers are solved using the custom **AHash** mining algorithm (70% weight).
   * Blocks must also be signed and validated by active stakers participating in validation (25% weight).
2. **Block Reward Distribution:**
   * **70% to Miners (17.5 ARU):** Rewards community hardware miners solving PoW equations.
   * **25% to Stakers (6.25 ARU):** Rewards validators securing the network through staking.
   * **5% to Treasury (1.25 ARU):** Automatically deposits protocol funds for long-term development.
3. **Fork Choice Rule:**
   The canonical chain is determined by a hybrid selection rule:
   1. The chain must consist of valid state transitions.
   2. The chain with the **Highest Accumulated Work (PoW weight)**.
   3. In case of identical work, the chain with the **Highest Finalized Stake Weight (PoS checkpoint finality)**.
   4. If still tied, the block accepted earliest by the local network interface.

### Staking Constraints:
* **Minimum Validator Stake:** 10,000 ARU.
* **Validator Offline Policy:** No slashing (no stake destruction). Validators lose block rewards while offline but can rejoin later.
* **Stake Domination Limit:** No single validator may control >10% of total staked weight to prevent validation monopolies.

## Alternatives
* **Alternative A: Pure Proof of Work (ASIC Resistance):** We rejected this because a pure PoW chain based on CPUs is highly susceptible to 51% rental attacks, and fails to provide finality guarantees.
* **Alternative B: Pure Proof of Stake (Delegated):** We rejected this because it favors early adopters and wealthy entities, locking out ordinary users who cannot afford large stakes.

## Consequences
* **Positive:**
  * **51% Attack Resistance:** An attacker must control both a majority of the CPU hashrate and a majority of the staked supply to successfully execute a double-spend or chain reorganization.
  * **ASIC Mitigation:** Stakers act as a validation gate. Even if ASICs are built, the stakers can reject blocks generated by centralized mining farms.
  * **Accessible Staking & Mining:** Users can stake, delegate, or run a light miner on Android smartphones without specialized hardware.
* **Negative:**
  * Hybrid verification increases the complexity of node block validation logic.
  * Synchronizing time and communication between stakers and miners requires robust P2P networking protocols.

## Migration
Not applicable. The hybrid consensus is active from the genesis block.

## Security Impact
The hybrid PoW + PoS consensus provides a zero-trust fallback. A block is only valid if it solves the AHash PoW equation *and* receives validation signatures from active validators. This double-gate mechanism protects the chain from long-range, 51%, and Sybil attacks.
