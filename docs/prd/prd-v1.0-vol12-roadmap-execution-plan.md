# EXECUTION PHILOSOPHY

ARUNA is designed and executed under these constraints:
* **Founder:** Solo Developer
* **Budget:** Near Rp 0
* **Team:** None
* **Infrastructure:** Homelab
* **Development Assistance:** AI Agents

### Key Rules:
* Build Small, Build Correctly, Build Consistently, Build Publicly.
* **Never build everything at once.**

---

# DEVELOPMENT STRATEGY (PROTOCOL FIRST)

Subsystems are built sequentially to manage complexity:
```
Consensus → Networking → State Engine → Wallet → Explorer → Governance → EVM → DEX
```

---

# SOLO FOUNDER STRATEGY

The founder wears multiple hats across lifecycle roles:
* **Protocol Architect:** Consensus, economics, and roadmap definitions.
* **Lead Engineer:** Rust development, workspace architecture, code reviews.
* **Community Builder:** Managing Discord/Telegram and public documentation.
* **Product Manager:** Defining PRD objectives, backlog, and release planning.

---

# AI AGENT STRATEGY

ARUNA is designed as an AI-native project. Development utilizes a dedicated agent squad:
1. **Chief Architect Agent:** Creates and updates PRD, ADRs, and structural frameworks.
2. **Protocol Engineer Agent:** Implements and refactors consensus, networking, and state transition logic.
3. **Rust Engineer Agent:** Core Rust code implementation, unit testing, and optimization.
4. **Security Auditor Agent:** Threat modeling, regression reviews, and vulnerability verification.
5. **Wallet Engineer Agent:** Flutter codebase implementation and wallet UI/UX.
6. **Explorer Engineer Agent:** Nuxt frontend, PostgreSQL database, and indexer syncing.

# 5-YEAR TESTNET & MAINNET TIMELINE

### Year 0: Foundation Phase (0–3 Months)
* **Objective:** Framework establishment.
* **Deliverables:** GitHub workspace setup, Architecture docs, ADR framework, Master PRD.
* **Success Criteria:** Protocol design frozen.

### Year 1: Sumatera Testnet
* **Objective:** Working Blockchain Prototype.
* **Milestones:**
  * **Q1:** Core Crates (`primitives`, `crypto`, `ahash`).
  * **Q2:** Networking (`libp2p` peer discovery and synchronization).
  * **Q3:** Consensus (`PoW`, `PoS`, and staking modules).
  * **Q4:** Node MVP (`aruna-node`).
* **Target:** 10 active nodes.

### Year 2: Kalimantan Testnet
* **Objective:** Public Integration Testing.
* **Milestones:**
  * **Q1:** Wallet MVP.
  * **Q2:** Explorer MVP.
  * **Q3:** Public RPC deployment.
  * **Q4:** Staking validator registry system.
* **Target:** 100 Nodes, 20 Validators.

### Year 3: Sulawesi Testnet
* **Objective:** Economic & Governance Validation.
* **Milestones:**
  * **Q1:** Governance system.
  * **Q2:** Treasury contracts.
  * **Q3:** Domain naming service.
  * **Q4:** Network stress testing.
* **Target:** 500 Nodes, 100 Validators.

### Year 4: Papua Release Candidate (RC)
* **Objective:** Mainnet Rehearsal.
* **Milestones:**
  * **Q1:** EVM MVP.
  * **Q2:** ARC-20 token standard.
  * **Q3:** DEX Testnet deployment.
  * **Q4:** Professional security audits.
* **Target:** 1,000 Nodes.

### Year 5: Jawa Mainnet
* **Objective:** Global Ecosystem Launch.
* **Launch checklist:**
  * Stable consensus, wallet, explorer, governance, treasury, security audit, and documentation.
* **Required Launch Targets:**
  * **1,000+** Nodes, **100+** Validators, and **10,000+** Wallets.
