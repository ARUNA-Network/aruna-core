Welcome & Vision

"Dari Rakyat. Oleh Rakyat. Untuk Rakyat. — Mine Anywhere. Owned By Everyone."

ARUNA Network is a community-owned, payment-optimized blockchain project originating from Indonesia, focused on ARM-centric computing, decentralized validator governance, and lightweight node accessibility.

Key Philosophy

  • Energy-conscious & CPU-friendly: Optimized directly for ARM64 servers, Raspberry Pi devices, mini PCs, and screen-off mobile mining.
  • Zero founder privileges: Clean architecture without founder privileges, hidden inflation minting, or private backdoors.
  • Stateless API access: Shielding physical validator node RPC interfaces using edge workers for caching and DDoS mitigation.

Core Consensus Specifications

ARUNA implements an Account-Based Ledger architecture utilizing RocksDB state database storage.

AHash Specification (PoW)

AHash is the PoW mining hash function designed specifically for ARM, CPU, and mobile-friendly mining.

AHash v1 Pipeline

Block Header -> BLAKE3 -> AES Mixing -> Argon2 Memory Expansion -> ARM NEON Opt -> Digest

Security Constraints

  • Thermal Throttling: Mobile miners must automatically throttle or pause execution if battery level drops below 20% or thermal limits are exceeded.
  • ASIC Resistance: Argon2 memory-hard expansion limits hashing parallelization inside custom chips (ASICs).

Cryptography Specifications

Proven cryptographic standards are employed throughout the ARUNA ledger core.

Standards

  • Signature Scheme: Ed25519 is utilized for fast consensus, validator signing, and wallets. secp256k1 is supported for EVM compatibility.
  • Hashing: BLAKE3 is the default internal hashing function.
  • Address Format: Bech32m format. Prefixes: sum1 (Sumatera), kal1 (Kalimantan), jaw1 (Jawa mainnet).

RPC API Reference

The Node RPC exposes diagnostic, blockchain inspect, and tx transaction posting routes.

Key Endpoints

  • GET /status: Returns node version, height, best block, sync status, and peer counts.
  • GET /live: Liveness check. Always returns 200 OK.
  • GET /ready: Readiness check. Verifies database access and initialization state.
  • GET /metrics: Telemetry stats formatted for Prometheus.
  • POST /tx: Submits a serialized transaction envelope to the mempool.

Explorer API Reference

Exposes paginated indexed ledger data reading from the PostgreSQL database.

Endpoints

  • GET /api/v1/blocks?limit=&offset=: Returns block summaries list.
  • GET /api/v1/transaction/:hash: Returns transaction detail parameters.
  • GET /api/v1/address/:address: Returns balance and transaction history.

ADR-0017: Explorer Architecture

Direct RocksDB reading by the public explorer is prohibited to avoid I/O bottlenecks on the validator node.

Instead, a dedicated indexer sidecar polls Node RPC endpoints and populates PostgreSQL, which is read exclusively by the REST API and explorer UI.

ADR-0018: Platform Evolution

Decouples ledger core from Edge workers and applications. Frontends must only connect to API layers or the Edge rpc-gateway proxy, completely shielding validator ports from direct public traffic.
